01
Create your bot with BotFather In progress

Every Telegram bot starts with BotFather — Telegram's official bot for creating bots. Open Telegram and search for @BotFather, then tap Start.

ℹ BotFather is the only official one — look for the blue verified checkmark next to its name.

Send BotFather the command /newbot. It will ask you two things:

Telegram chat
You: /newbot

BotFather: Alright, let's create your bot. What name
           will it have? (This is the display name.)

You: My First Bot

BotFather: Good. Now let's choose a username for
           your bot. It must end in `bot`.

You: myfirstbot_bot

BotFather: Done! Here is your token:
           1234567890:ABCDefGhIJklmNoPQRstuVWxyZ
           Keep it safe — this is your bot's password.
⚠ Never share your token publicly or commit it to GitHub. Anyone with your token controls your bot.

Copy that token somewhere safe. You'll need it in step 3.

02
Set up Python and install the library Todo

You'll write your bot in Python using the python-telegram-bot library. First make sure Python is installed — open your terminal and check:

Terminal 
python --version

If you see Python 3.x.x you're good. If not, download Python from python.org. Then create a folder for your project and install the library:

Terminal 
mkdir my-telegram-bot
cd my-telegram-bot
pip install python-telegram-bot
ℹ On some systems you may need pip3 instead of pip, and python3 instead of python.
03
Write your first bot Todo

Create a new file called bot.py in your project folder and paste this code in. This bot will respond to /start and /hello, and echo back anything else you send it.

Python — bot.py 
from telegram import Update
from telegram.ext import ApplicationBuilder, CommandHandler, MessageHandler, filters, ContextTypes

# Replace this with your token from BotFather
TOKEN = "YOUR_TOKEN_HERE"

# /start command
async def start(update: Update, context: ContextTypes.DEFAULT_TYPE):
    await update.message.reply_text(
        "👋 Hello! I'm your first Telegram bot.\n"
        "Try sending /hello or just type anything!"
    )

# /hello command
async def hello(update: Update, context: ContextTypes.DEFAULT_TYPE):
    name = update.effective_user.first_name
    await update.message.reply_text(ff"Hey {name}! Nice to meet you 🤖")

# Echo handler — replies with whatever the user sends
async def echo(update: Update, context: ContextTypes.DEFAULT_TYPE):
    user_msg = update.message.text
    await update.message.reply_text(ff"You said: {user_msg}")

# Build and start the bot
app = ApplicationBuilder().token(TOKEN).build()

app.add_handler(CommandHandler("start", start))
app.add_handler(CommandHandler("hello", hello))
app.add_handler(MessageHandler(filters.TEXT & ~filters.COMMAND, echo))

print("Bot is running... Press Ctrl+C to stop.")
app.run_polling()
⚠ Replace "YOUR_TOKEN_HERE" with the actual token you got from BotFather in step 1.
04
Run your bot Todo

In your terminal, make sure you're inside the my-telegram-bot folder, then run:

Terminal 
python bot.py

You should see Bot is running... printed. Now open Telegram, search for your bot's username (the one you chose in step 1), and tap Start. Try sending it /hello or any message!

✓ The bot only works while this terminal is open and running. To stop it, press Ctrl + C.
ℹ Right now the bot runs on your computer. To keep it running 24/7, you'd deploy it to a server — like Azure! That's a great next step.
05
Keep your token safe with .env Todo

Hardcoding your token in bot.py is fine for testing, but dangerous if you push to GitHub. The proper way is to store it in a .env file that you never commit.

First install the helper library:

Terminal 
pip install python-dotenv

Create a file named .env in your project folder:

.env 
BOT_TOKEN=1234567890:ABCDefGhIJklmNoPQRstuVWxyZ

Create a .gitignore file to prevent the token from being committed:

.gitignore 
.env
__pycache__/

Then update the top of bot.py to load from the file:

Python — top of bot.py 
import os
from dotenv import load_dotenv
load_dotenv()

TOKEN = os.getenv("BOT_TOKEN")
✓ Now your token is safe. You can push bot.py and .gitignore to GitHub — but never .env.
